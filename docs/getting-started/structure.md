## Specs Directory

### Status Directories

Features move through status directories as they progress:

| Directory | Description |
|-----------|-------------|
| `backlog/` | Planned features, not yet started |
| `active/` | Features currently being implemented |
| `completed/` | Shipped features with release info |
| `archived/` | Cancelled or superseded features |

### Feature Directory

Each feature has its own directory containing all related specs:

```
specs/active/user-authentication/
в”њв”Ђв”Ђ mrd.json         # Market Requirements Document
в”њв”Ђв”Ђ prd.json         # Product Requirements Document
в””в”Ђв”Ђ evaluation.json  # Spec evaluation results
```

### Constitution

The `constitution.json` file defines organization-wide standards:

```
specs/constitution.json
```

It covers:

- Tech stack (languages, frameworks, databases)
- Architecture standards
- File organization rules
- Quality standards
- Security requirements
- Process standards

### Roadmap

The `roadmap.json` file tracks all features by status:

```
specs/roadmap.json
```

Structure:

```json
{
  "version": "1.0.0",
  "updated_at": "2025-01-15T00:00:00Z",
  "active": [
    {"feature": "user-auth", "priority": 1, "owner": "Team A"}
  ],
  "backlog": [
    {"feature": "data-export", "priority": 1, "blocked_by": ["user-auth"]}
  ],
  "completed": [],
  "archived": []
}
```

### Index Generation

The `dfspec generate index` command creates index files:

```bash
# Generate for all features
dfspec generate index

# Generate for specific feature
dfspec generate index user-authentication
```

## Naming Conventions

### Feature Names

- Use `kebab-case`
- Descriptive but concise
- Examples: `user-authentication`, `data-export`, `query-caching`

### Version Format

- Semantic versioning: `vX.Y.Z`
- Examples: `v1.0.0`, `v2.1.3`

### Spec IDs

- Format: `{TYPE}-{YEAR}-{NUMBER}`
- Examples: `MRD-2025-001`, `PRD-2025-001`, `FR-001`

## Spec-to-Doc Mapping

Specs in JSON format can be mapped to docs in Markdown:

| Spec File | Doc File |
|-----------|----------|
| `specs/active/user-auth/mrd.json` | `docs/design/user-auth/mrd.md` |
| `specs/active/user-auth/prd.json` | `docs/design/user-auth/prd.md` |

The mapping strategy is configurable in the constitution:

```json
{
  "file_organization": {
    "spec_to_doc_mapping": {
      "strategy": "feature",
      "doc_file_extension": ".md",
      "generate_index": true
    }
  }
}
```

## Generated Files

Some files are generated and should not be edited manually:

| File | Generated By |
|------|--------------|
| `ROADMAP.md` | `dfspec generate roadmap` |
| `docs/design/{feature}/index.md` | `dfspec generate index` |
| `schemas/*.schema.json` | `go run cmd/generate-schema/main.go` |
